Проект “Замена SWIFT”
Цель проекта
Целью проекта “Замена SWIFT для цифровой экосистемы” (или сокращенно проект “Замена SWIFT”) является создание и ввод в эксплуатацию цифровой инфраструктуры на основе налично-подобных (на предъявителя) «звонких» цифровых денег для обеспечения оптовых трансграничных и трансвалютных цифровых транзакций в группе стран (цифровой экосистемы) через их коммерческие банки-операторы. Проект “Замена SWIFT” является составной частью направления “Цифровые деньги и цифровые платежи” (код “DFMI”) Программы ЦТМ.

Проект “Замена SWIFT” представляет собой переходное решение, построенное на основе стандартного функционала коммерческого коробочного продукта “Цифровая Инфраструктура Финансового Рынка» (ЦИФР или DFMI).
• GREENДля осуществления трансграничных платежей продукт DFMI использует: • GREENалгоритмический платежный процесс ES controller; • GREENалгоритмический брокер для обмена валюты Market Maker; • GREENуникальные защищённые идентификаторы SIN.

Потоки ценностей между СТАРОЙ и НОВОЙ финансовыми системами
Предупреждение: Проект “Замена SWIFT” должен балансировать между СТАРОЙ финансовой системой и НОВОЙ финансовой системой без изменения СТАРОЙ финансовой системы.
В каждой стране NNN будет выбран один дружественный коммерческий банк (банк-оператор), который будет функционировать в качестве брокера для двунаправленного обмена 1:1 между деньгами старой финансовой системы (FIAT-NNN) и деньгами из новой финансовой системы (LCU-FIAT-NNN).
Банк-оператор и компания KUT DFMI заключают контракт, по которому банк-оператор получает доступ к DFMI страны. Банк-оператор приобретает необходимые активы, т.е. WCU для использования DFMI страны.
Прямой связи между старой финансовой системой и новой финансовой системой нет, поэтому передача информации строится на логике “перекинуть через стену” и “воздушный зазор”.

Описание трансграничных платежей в проекте “Замена SWIFT”
Плательщик / Покупатель / ZZZ из страны ААА и Получатель платежа / Продавец / YYY из страны ССС ведут дела в соответствии с обычной деловой практикой, как показано ниже:
1. Плательщик / Покупатель / ZZZ отправляет Получателю платежа / Продавцу / YYY заказ на покупку.
2. Получатель платежа / Продавец / YYY отправляет Плательщику / Покупателю счет PayMe с суммой в деньгах своей страны (FIAT-CCC).
3. Плательщик / Покупатель / ZZZ отправляет Получателю платежа / Продавцу / YYY платеж в виде автономной (оффлайн) цифровой монеты с соответствующей суммой в WCU.
4. Получатель платежа / Продавец / YYY отправляет Плательщику / Покупателю / ZZZ подтверждение платежа.


Для простоты ниже поясняется только пункт 3 из этого списка.
Считается, что страна ААА использует фиатные деньги FIAT-AAA и их цифровую версию LCU-FIAT-AAA. Плательщик / Покупатель / ZZZ имеет счет BBAN ZZZ в коммерческом банке AAA в стране AAA (это банк-оператор AAA).

Страна CCC использует фиатные деньги FIAT-CCC и их цифровую версию LCU-FIAT-CCC. Получатель платежа / Продавец / YYY имеет счет BBAN YYY в коммерческом банке CCC в стране CCC (это банк-оператор CCC).
Банк-оператор ААА (который выступает в качестве брокера между старой финансовой системой и новой финансовой системой) имеет SIN в DFMI-AAA:
“BANK ААА” с LCU-FIAT-AAA и WCU (это цифровые активы банка ААА).

Банк-оператор CCC (который выступает в качестве брокера между старой финансовой системой и новой финансовой системой) имеет SIN в DFMI-CCC:
“BANK CCC” с LCU-FIAT-CCC и WCU (это цифровые активы банка ССС).

На пилотной стадии проекта используются сервисы “KUT DFMI” в стране. Эти сервисы будут исполнять переток ценности: сначала автоматизировано, а потом автоматически. После этого такое исполнение может быть передано Глобальному Цифровому Банку BFi. Текст ниже составлен с использованием таких сервисов “KUT DFMI”.
Переток ценности от Плательщика / Покупателя / ZZZ к Получателю платежа / Продавцу YYY выглядит следующим образом:
1. AAA чтобы переслать какую-то сумму старых денег (FIAT-AAA) со своего счета BBAN ZZZ в банке-операторе ААА Получателю платежа / Продавцу / YYY в стране ССС.
2. AAA запрашивает банк-оператор ААА дебетировать счет BBAN ZZZ на указанную сумму старых денег (FIAT-AAA).
3. AAA использует в DFMI-AAA такое же количество LCU-FIAT-AAA для SIN BANK AAA и конвертирует их в WCU в SIN BANK AAA.
4. AAA создает в DFMI-AAA оффлайн монету в WCU из SIN BANK AAA, который дебетируется.
5. AAA отправляет оффлайн монету в WCU к сервису KUT DFMI CCC.
6. CCC вносит полученную оффлайн монету в WCU в DFMI-CCC в SIN BANK CCC, который кредитуется.
7. CCC конвертирует полученную монету в LCU-FIAT-CCC в SIN BANK CCC и информирует банк-оператор CCC о получении платежа для Получателя платежа / Продавца / YYY от Плательщика / Покупателя / ZZZ.
8. Банк-оператор CCC зачисляет на счет BBAN YYY полученную сумму в старых деньгах (FIAT-CCC).
9. Получатель платежа / Продавец / YYY информируется о получении старых денег (FIAT-CCC) от Плательщика / Покупателя / ZZZ.

Это начальная схема, которая может быть адаптирована во время пилотирования. Со временем KUT DFMI сервис будет выполняться Глобальным Цифровым Банком BFi.

Требования к банкам-операторам

Банк-оператор оказывает эксклюзивную услугу KUT DFMI, включая контроль трансграничных платежей и контроль ликвидности.

Налоговый агент

Считается, что расчет налога, подлежащего уплате, будет осуществляться налоговым агентом для каждой страны на основе данных о трансграничных платежах.

Техническая схема решения “Замена SWIFT”

В начале проекта экосистема использует одну общую инфраструктуру с:
1. один экземпляр ЦИФР для каждой страны с удаленным доступом из Центрального Банка
2. экземпляр ЦИФР, который выступает в качестве «Инновационного хаба» для экосистемы.
3. экземпляр ЦИФР из отдельной инфраструктуры, который используется для демонстрационных целей. Это DEMO экземпляр ЦИФР.

Этапы проекта “Замена SWIFT”
Для проекта предлагается несколько этапов (групп задач); при этом, некоторые этапы и задачи могут перекрываться по времени.

Этап 1: Обучение
Цель 1: Достижение хорошего общего понимания между всеми заинтересованными сторонами по проекту, процессу оплаты и вспомогательным процессам.
Задача 1A: Создание экземпляра ЦИФР “Инновационный хаб” для экосистемы.
Задача 1B: Построение DEMO-экземпляра ЦИФР.
Задача 1С: Объяснение и демонстрация ЦИФР заинтересованным сторонам.

Этап 2: Пилот
Цель 2: Достижение договоренности о решении для трансграничных и трансграничных оптовых платежей.
Задача 2A: Создание экземпляров ЦИФР для каждой страны в цифровой экосистеме.
Задача 2В: Построение приложения для трансграничных и трансвалютных оптовых платежей.
Задача 2С: Объяснить и продемонстрировать решение заинтересованным сторонам.
Задача 2D: Создание дополнительных (вспомогательных) приложений и сервисов (например, службы поддержки).
Задача 2Е: Дать заинтересованным сторонам попробовать приложение.

Этап 3: Опытно-промышленная эксплуатация
Цель 3: Стабилизировать решение.
Задача 3A: Поддержка стран и обновление приложений при необходимости.
Задача 3В: Создать и ввести в действие Международную ассоциацию расчетов (ISA) – это замена BIS.

Этап 4: Построение автономной версии ЦИФР
Цель 4: Достижение решения, которое не подвержено санкциям.
Задача 4A: Разработать и сертифицировать собственный HSM.
Задача 4B: Создание автономного ЦИФР версии 1.
Задача 4C: Создание автономного ЦИФР версии 2.

Этап 5: Создание производственной среды

Этап 6: Производственные операции с горячей поддержкой

Этап 7: Постоянная поддержка
